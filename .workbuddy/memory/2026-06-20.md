## 2026-06-20（晚）全屏回滚

### 背景
全屏布局改动（按钮移到下面、加进度条、设置面板等）经过多次迭代都没有在手机上生效（原生全屏走的是 MainActivity.java，改 CSS/JS 无效）。用户反馈"还不如原来没改的版本"。

### 回滚操作
- 从 commit `6698591` 恢复了 `app.js`、`style.css`、`MainActivity.java`（全屏按钮回到顶部原始布局）
- **保留** `widescreen.css` 的热门推荐横屏5列修复（`minmax(0, 1fr)`）
- **保留** `index.html` 的版本号显示
- 版本号更新到 v3.9.4，缓存参数同步更新
- commit `c2c8685` 已推送

### 教训
- Android 原生全屏（`onShowCustomView`）由 `MainActivity.java` 控制，CSS/JS 的 `applyFullscreenCSS()` 只在非 Android 环境生效
- 改原生全屏需要改 Java 代码并重新编译 APK，改 CSS/JS 只能影响 WebView 内的样式
- 下次改全屏 UI 必须直接改 `MainActivity.java`，并在本地构建测试，不能靠 GitHub Actions
