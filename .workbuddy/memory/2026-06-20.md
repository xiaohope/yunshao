# 2026-06-20 工作日志

## 框架重构：统一为 CSS 全屏方案

### 问题
- 用户反馈：全屏播放进度条不更新、视频标题不显示、布局还是旧版
- 根本原因：项目同时存在 CSS 全屏和 Android 原生全屏两套方案，之前一直改的是 CSS 全屏代码，但手机上走的是原生全屏，改动根本不生效

### 决策：只用 CSS 全屏，删掉原生全屏
- 原因：所有 UI（进度条、按钮、标题）都在 HTML/JS 里，维护两套太痛苦
- 方案：CSS 全屏 + 通过 JS 桥调用原生 `hideSystemUI()`/`showSystemUI()` 隐藏/恢复系统栏

### 改动内容

#### Java (MainActivity.java)
- `YunShaoNative` JS 接口加了两个方法：`hideSystemUI()` / `showSystemUI()`
- `onShowCustomView()` 禁用（直接 return，不再触发原生全屏）
- `showSystemBars()` 修复：清除 `FLAG_FULLSCREEN` 和 `FLAG_LAYOUT_NO_LIMITS`，否则退出全屏后状态栏不恢复

#### JS (app.js)
- `enterFullscreenMode()`：统一调用 `applyFullscreenCSS()`，同时调 `YunShaoNative.hideSystemUI()`
- `exitFullscreenMode()`：统一调用 `removeFullscreenCSS()`，同时调 `YunShaoNative.showSystemUI()`
- CSS 全屏控制层加了进度条 + 时间显示，支持点击跳转
- 加了 500ms 定时器实时更新进度条

#### CSS (style.css)
- 加了 `.fs-progress-bar` / `.fs-progress-track` / `.fs-progress-played` 等样式
