## 普通视频全屏原生进度条不隐藏 v3.17.1

### 问题描述
TV 直播全屏时原生进度条已成功隐藏，但普通视频全屏时原生进度条仍然显示。

### 诊断过程 (mp-diagnosing-bugs)
对比 `playCurrent()`（普通视频）和 TV 直播的视频创建方式，发现两个关键差异：

**根因1**: 普通视频创建时有 `pointer-events:none`（line 1926）
- TV 视频无此属性 → `touchstart preventDefault` 正常触发 → Android WebView 原生控件被阻止
- 普通视频有 `pointer-events:none` → `touchstart` 事件穿透，`preventDefault()` 永远不触发
- `applyFullscreenCSS` 中添加的 `video.addEventListener("touchstart", ev.preventDefault())` 形同虚设

**根因2**: `_fsBottomMask`（65px黑色底部遮罩）创建后从未 `appendChild` 到 DOM
- 注释写"先添加到pa"，但缺失了 `pa.appendChild(fsBottomMask)` 调用
- `removeFullscreenCSS` 中有 `querySelector('._fsBottomMask')` 尝试移除，但从未添加过
- 注释里的"之后会移到controls内部"也未实现

### 修复 (commit e9b8f72)
1. `applyFullscreenCSS`: 添加 `pa.appendChild(fsBottomMask)` — 遮罩现在真正渲染
2. `applyFullscreenCSS`: 添加 `video.style.pointerEvents = 'auto'` — 覆盖普通视频的 `pointer-events:none`
3. `removeFullscreenCSS`: 退出全屏时恢复 `video.style.pointerEvents = 'none'`
4. 版本: v3.17.1 (versionCode 53)

## 真正根因：X5内核原生全屏播放器 v3.17.2

### 用户追加反馈
"原生播放器有个全屏模式，显示进度条一会，就进入沉浸式观看就会隐藏"

### 重新诊断
这个行为是X5(TBS)内核原生播放器的特征。检查发现普通视频 line 1928：
```javascript
video.setAttribute('x5-video-player-fullscreen','true');
```
X5内核看到这个属性会显示自己的原生播放器UI（进度条/控制栏），
显示数秒后进入沉浸模式自动隐藏。

X5内核**忽略 `video.controls` 属性**，只认 `x5-video-player-fullscreen` 属性。

TV直播未复现的原因：X5对HLS流(.m3u8)与普通视频(.mp4)的处理不同，
前者可能不触发原生播放器覆盖层。

### 修复 (commit 3db9667)
1. `enterFullscreenMode`: 设置 `v.setAttribute('x5-video-player-fullscreen', 'false')` (2处: line 214+910)
2. `applyFullscreenCSS`: 设置 `video.setAttribute('x5-video-player-fullscreen', 'false')`
3. `removeFullscreenCSS`: 退出时恢复 `v.setAttribute('x5-video-player-fullscreen', 'true')`
4. 版本: v3.17.2 (versionCode 54)
5. push 成功 (e9b8f72 + 3db9667 → GitHub Actions 自动构建)

### v3.17.1 的改动也包含在内
- fsBottomMask 实际 appendChild（65px黑色底部遮罩）
- pointer-events auto/none 切换（辅助措施，增强触摸事件处理）
